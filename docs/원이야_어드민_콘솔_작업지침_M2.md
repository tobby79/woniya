# 원이야 어드민 콘솔(admin.html) 개편 작업 지침 (M2)

> 목적: 원장이 DB를 직접 건드리지 않고, 콘솔에서 현재 템플릿의 **모든 섹션**을 편집할 수 있게 한다.
> 원칙: 탭 구조 / 사진은 해당 섹션에 인라인 업로드 / 현재 4개 템플릿이 렌더하는 모든 데이터를 빠짐없이 편집 가능하게.

---

## 0. 작업 방식 (중요)

- **탭 껍데기부터 만들고, 탭 하나씩 순서대로 구현**한다. 한 번에 전부 만들지 않는다.
- 각 탭 완료 시 로컬 확인 → 문제 없으면 다음 탭.
- 각 탭(또는 2개 단위) 완료 시 커밋.
- 기존 admin.html의 인증 흐름(로그인/로그아웃/세션), 사진 업로드 유틸(compressImage, signedUrl, handleFiles), 상담 목록 로직은 **재사용**한다. 새로 짜지 않는다.

---

## 1. 전체 탭 구조 (10개 → 묶어서 정리)

| # | 탭 | 편집 대상 | DB 위치 |
|---|---|---|---|
| 1 | 기본 정보 | 원 이름·주소·운영시간·소개 + 히어로 사진 | centers (name, address, operating_hours, intro) + center_media(hero) |
| 2 | 원 소개 | 원장 인사말+사진 / 원훈·교육가치 | centers.director(jsonb) + centers.philosophy(jsonb) + center_media(director 사진) |
| 3 | 하루 이야기 | 시간대별 스토리 5개 | center_media(day_story) + 각 row의 메타 |
| 4 | 시설·프로그램 | 시설 현황+항목+사진 / 교육프로그램 항목+사진 | centers.facilities(jsonb) + centers.programs(jsonb) + center_media |
| 5 | 선생님 | 선생님 카드(이름·역할·한마디·사진) | center_media(teacher) + 각 row 메타 |
| 6 | 입학안내 | 모집 대상·정원·기간·절차·준비물·기타 | admissions 테이블 |
| 7 | 오시는 길 | 주소·교통편 텍스트 / 정류장·시간(추후) | centers.directions_text + (버스 노선은 별도 작업) |
| 8 | FAQ | 질문·답변 추가·편집·삭제 | centers.faqs(jsonb) |
| 9 | 신청 현황 | 상담·대기 신청 + 입소 상담 목록 (읽기 전용) | consultations 테이블 |

> 참고: hero/notices/schedule/finale/footer/menu/badges/tags 등은 이번 콘솔 편집 범위에서 **제외**(추후 또는 자동 생성). 단 hero 사진은 1번 탭에서 업로드 가능.

## 2. DB 구조 참고 (실제 컬럼)

### centers (편집 대상 컬럼만)
- name (text), address (text), operating_hours (text), intro (text), directions_text (text)
- director (jsonb): { badge, photo, photo_alt, message, sign_label, sign_name, scroll_cue }
- philosophy (jsonb): { greeting, motto, values[] }
- programs (jsonb): { intro, items[ {name, desc} ] }
- facilities (jsonb): { intro, stats[ {label, value} ], items[ {name, desc} ] }
- faqs (jsonb): { eyebrow, title, items[ {q, a} ] }

### admissions
- center_id (uuid), target, capacity_info, period, process, supplies, notes (모두 text)

### center_media
- center_id (uuid), media_type (hero/day_story/album/teacher/director), photo_url (storage path), caption, sort_order
- day_story row 메타: slot, time_label, title, photo_alt, caption, note
- teacher row 메타: title(이름), subtitle(역할), caption(한마디), photo_url

### consultations
- center_id, parent_name, phone, kind(consult/waiting/info_session), wish_class, wish_time, consent_at, created_at

## 3. 탭별 상세 명세

### 탭 1. 기본 정보
- 입력 필드: 원 이름(name), 주소(address), 운영시간(operating_hours), 소개(intro)
- 히어로 사진: center_media media_type='hero' (최대 1장). 기존 업로드 UI 재사용, 타입 고정.
- 저장: centers UPDATE.

### 탭 2. 원 소개
- **원장 인사말** (centers.director jsonb):
  - badge, message(textarea), sign_label, sign_name
  - 원장 사진: **현재는 A 방식 확정** — 사진 업로드 후 storage 경로를 director.photo에 문자열로 저장(center_media 행으로 만들지 않음). 템플릿 렌더는 director.photo를 그대로 읽으므로 변경 없음.
  - (추후 B 방식으로 통일 예정: center_media media_type='director'로 이관 + 템플릿 4개 렌더 로직 수정. 이번 범위 아님.)
- **원훈·교육가치** (centers.philosophy jsonb):
  - greeting(textarea), motto(text), values(태그 입력 — 여러 개)
- 저장: centers.director, centers.philosophy 각각 jsonb로 UPDATE.

### 탭 3. 하루 이야기
- center_media media_type='day_story' 행들을 카드 리스트로.
- 각 카드: 시간 라벨(time_label), 제목(title), 설명(note), 캡션(caption), 사진(photo_url), slot
- 추가/편집/삭제 + 순서(sort_order)
- 최대 5개 권장(제한은 기존 LIMITS 참고).

### 탭 4. 시설·프로그램
- **시설 현황** (facilities.stats): label/value 쌍 추가·편집·삭제 (예: 설립연도/2012년)
- **시설 항목** (facilities.items): name/desc 추가·편집·삭제
- **시설 intro** (facilities.intro): textarea
- **교육프로그램 intro** (programs.intro): textarea
- **교육프로그램 항목** (programs.items): name/desc 추가·편집·삭제
- 사진: 현재 시설/프로그램은 album 이미지를 순서 매칭해서 빌려쓰는 중. 별도 사진 업로드는 추후(템플릿 만들기 단계). 이번엔 텍스트 편집만, album 사진은 기존 앨범 업로드로.
- 저장: centers.facilities, centers.programs jsonb UPDATE.

### 탭 5. 선생님
- center_media media_type='teacher' 행들을 카드 리스트로.
- 각 카드: 이름(title), 역할(subtitle), 한마디(caption), 사진(photo_url)
- 추가/편집/삭제 + 순서

### 탭 6. 입학안내
- admissions 테이블 (center당 1행 가정).
- 필드: target(모집 대상), capacity_info(정원), period(기간), process(절차), supplies(준비물), notes(기타)
- 행 없으면 INSERT, 있으면 UPDATE.

### 탭 7. 오시는 길
- centers.directions_text (교통편 텍스트, textarea)
- 정류장·시간/버스 노선 정보: **이번 범위 제외** (별도 작업 예정). UI에 "추후 제공" 안내만.

### 탭 8. FAQ
- centers.faqs jsonb의 items[] (q, a 쌍) 추가·편집·삭제.
- eyebrow, title은 고정값 유지 가능.

### 탭 9. 신청 현황 (읽기 전용)
- consultations 목록. 기존 loadConsultations 재사용.
- kind별 라벨(상담/대기/설명회) 유지.

---

## 4. UI/UX 규칙

- 탭 바: 상단 sticky, 가로 스크롤 가능(모바일). 현재 탭 강조.
- 각 탭은 독립 패널. 저장 버튼은 탭별로.
- 사진 업로드: 기존 드래그앤드롭 UI 재사용, 해당 섹션에 인라인 배치(별도 사진 탭 없음).
- jsonb 배열(values, stats, items, faqs, day_story, teacher) 편집은 "항목 추가/삭제 + 인라인 입력" 패턴 통일.
- 저장 결과 메시지(성공/실패) 탭별 표시.
- 디자인: 기존 admin.html 톤(크림/코랄/Pretendard) 유지.

---

## 5. 기술 주의사항

- jsonb 저장 시 기존 구조 유지(부분 필드만 덮어쓰지 말고 객체 전체 구성 후 UPDATE).
- center_media 메타 편집은 행 단위 UPDATE.
- RLS: 원장은 본인 원(owner_id) 데이터만 접근. 정책 확인 필요.
- 사진 업로드 경로: {center_id}/{type}/{timestamp}_{filename} (기존 패턴 유지).
- service_role 키 노출 금지(기존대로 anon + RLS).

## 6. 작업 순서 (권장)

1. 탭 껍데기(탭 바 + 패널 전환) + 탭1(기본 정보) → 커밋
2. 탭2(원 소개) → 커밋
3. 탭3(하루 이야기) → 커밋
4. 탭4(시설·프로그램) → 커밋
5. 탭5(선생님) → 커밋
6. 탭6(입학안내) + 탭7(오시는 길) → 커밋
7. 탭8(FAQ) + 탭9(신청 현황) → 커밋

각 단계 로컬 확인 후 진행.
